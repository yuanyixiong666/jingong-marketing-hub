## 踩坑与调试

### 1. Qwen3 思考模式空响应
- **现象**：LLM 返回 content 为空
- **原因**：Qwen3 默认开启思考模式，reasoning_content 消耗了 max_tokens
- **解决**：payload 添加 `enable_thinking: False`
- **发现者**：人工调试

### 2. enable_thinking 参数位置
- **现象**：参数不生效
- **原因**：httpx 直接调用不支持 OpenAI SDK 的 extra_body
- **解决**：将参数放在 payload 顶层
- **发现者**：人工调试

### 3. 任务列表 500 错误
- **现象**：GET /api/tasks 返回 500
- **原因**：SQLAlchemy ORM 对象无法被 Pydantic 直接序列化
- **解决**：引入 CrawlTaskOut Schema 做转换
- **发现者**：AI 建议方案，人工确认

### 4. 微信小程序请求超时
- **现象**：前端 API 请求全部超时
- **原因**：微信开发者工具默认校验域名合法性
- **解决**：开启"不校验合法域名"选项
- **发现者**：人工发现

## 总结

本项目在 3 天内完成了从项目骨架到功能完整的数字营销数据中台。AI 在代码生成阶段贡献了约 95% 的初始代码，人工工作集中在三个方向：业务领域知识注入（调味品行业 Prompt、模拟数据）、技术调试（LLM 参数、序列化 Bug）、以及前端交互打磨。
